## 项目概述

### 背景
- 个人练手项目，用于提升技术能力和面试展示
- 为家人企业搭建官网，展示公司近况
- 同时作为个人博客平台，发布技术文章
- 已有两个参考项目（website-builder快速版和Java版）

### 目标
- **短期目标**：实现个人博客功能，能够发布和展示文章
- **中期目标**：完善企业官网功能，支持自定义页面和样式
- **长期目标**：打造一个可复用的CMS系统，作为面试作品展示
- **成功标准**：功能完整、代码规范、可演示、能体现技术深度

### 用户画像
| 用户类型 | 特征 | 痛点 | 期望 |
|---------|------|------|------|
| 内容管理员 | 非技术人员，需要简单易用的后台 | 不懂代码，需要可视化编辑 | 像使用Word一样编辑网页 |
| 企业访客 | 潜在客户、合作伙伴 | 想快速了解企业信息 | 页面美观、信息清晰 |
| 博客读者 | 技术爱好者、同行 | 寻找有价值的技术内容 | 阅读体验好、内容有深度 |
| 面试官 | 技术面试官 | 评估候选人技术能力 | 代码质量高、架构合理、有亮点 |

## 核心功能细化清单

### 模块A：用户认证与权限管理

#### A1. 用户登录
| 细化点 | 详细描述 | 验收标准 |
|--------|----------|----------|
| A1.1 登录页面 | 包含用户名、密码输入框，记住我选项，登录按钮 | 页面布局美观，表单验证正确 |
| A1.2 账号密码验证 | 后端验证用户名密码，密码使用BCrypt加密 | 错误3次锁定账号5分钟 |
| A1.3 JWT Token生成 | 登录成功后生成JWT，包含用户ID、角色、过期时间 | Token有效期24小时，可刷新 |
| A1.4 登录状态保持 | 前端存储Token到localStorage，自动附加到请求头 | 刷新页面保持登录状态 |
| A1.5 登录失败提示 | 显示友好的错误提示（用户名或密码错误） | 不泄露具体是用户名还是密码错误 |
| A1.6 退出登录 | 清除Token，跳转到登录页 | 退出后无法访问需要认证的页面 |

#### A2. 权限控制
| 细化点 | 详细描述 | 验收标准 |
|--------|----------|----------|
| A2.1 角色定义 | 定义管理员、编辑、访客三种角色 | 数据库有角色表，支持扩展 |
| A2.2 接口权限拦截 | 后端使用拦截器验证Token和权限 | 无权限返回403状态码 |
| A2.3 前端路由守卫 | 前端根据角色控制页面访问 | 无权限自动跳转到403页面 |
| A2.4 按钮级权限 | 根据角色显示/隐藏操作按钮（如删除） | 编辑角色看不到删除按钮 |

### 技术难点和解决方案

#### 1. 菜单数量限制实现

**方案**：三层防护

- 前端：实时计算菜单数，达到10个时禁用添加按钮
- 后端：接口层校验，返回错误码
- 数据库：通过触发器或应用层事务控制

#### 2. 文章页面动态布局

**方案**：后端提供菜单层级信息

```typescript
// 后端返回文章时附带菜单信息
GET /api/articles/:id
Response: {
  article: {...},
  menuInfo: {
    level: 1 | 2 | 3,
    hasChildren: boolean,
    siblings: Menu[], // 同级菜单
    children: Menu[]  // 子菜单
  }
}

// 前端根据 menuInfo.level 决定布局
if (menuInfo.level === 1 && !menuInfo.hasChildren) {
  // 单级菜单：左侧显示标签
  layout = 'tags-left';
} else if (menuInfo.level > 1 || menuInfo.hasChildren) {
  // 多级菜单：右侧显示导航
  layout = 'nav-right';
}
```
