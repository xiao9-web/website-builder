# AI团队工作流模板

## 📋 完整开发流程

### 阶段1：需求分析
```
/analyze-demand
```
- 理解用户需求
- 拆解任务
- 评估风险
- 输出：需求分析报告

### 阶段2：产品设计
```
/create-prd
```
- 创建产品需求文档
- 定义功能范围
- 确定验收标准
- 输出：PRD文档

### 阶段3：架构设计
```
/design-arch
```
- 技术选型
- 架构设计
- 接口设计
- 输出：架构设计方案

### 阶段4：开发实现
```
/dev
```
- 编写代码
- 单元测试
- 代码格式化
- 输出：可运行代码

### 阶段5：代码审查
```
/code-review
```
- 质量检查
- 安全审查
- 性能评估
- 输出：审查报告

### 阶段6：优化改进
```
/optimize
```
- 性能优化
- 代码重构
- 最佳实践应用
- 输出：优化后的代码

### 阶段7：测试验证
```
/test
```
- 单元测试
- 集成测试
- 端到端测试
- 输出：测试报告

### 阶段8：安全审计
```
/security
```
- 安全扫描
- 漏洞检测
- 合规检查
- 输出：安全报告

### 阶段9：部署上线
```
/deploy
```
- 构建打包
- 部署配置
- 监控告警
- 输出：部署文档

## 🔄 快速工作流

### 快速修复流程
```
/quick-fix → /test → /code-review
```

### 新功能开发流程
```
/analyze-demand → /design-arch → /dev → /test → /code-review → /deploy
```

### 文章写作流程
```
/write-article → /code-review (检查代码示例)
```

### 代码优化流程
```
/context-prime → /optimize → /test → /code-review
```

## 💡 使用技巧

1. **串联使用**：复杂任务可以串联多个命令
2. **并行处理**：独立任务可以同时进行
3. **迭代优化**：根据反馈不断改进
4. **保存记忆**：重要决策会自动保存
