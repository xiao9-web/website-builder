
# 成本优化指南 - 文档4.3实现

## 成本计算公式
```
单次对话成本 = (输入tokens / 1M * 输入价格) + (输出tokens / 1M * 输出价格)
```

### 模型价格表（2026年）
| 模型 | 输入价格（美元/百万tokens） | 输出价格（美元/百万tokens） | 适用场景 |
|------|-----------------------------|------------------------------|----------|
| Claude Haiku 4.5 | 0.25 | 1.25 | 简单代码生成、日常问答、格式化输出 |
| Claude Sonnet 4.6 | 3.0 | 15.0 | 代码审查、架构设计、复杂推理 |
| Claude Opus 4.5 | 15.0 | 75.0 | 高级架构设计、安全审计、复杂问题解决 |

## 成本优化策略
### 策略1：选择合适的模型
**按任务类型选择模型**：
- 简单任务（代码格式化、简单修改、文档生成）：使用 Haiku 模型，成本仅为Sonnet的1/12
- 中等任务（代码审查、API开发、常规需求）：使用 Sonnet 模型，平衡质量和成本
- 复杂任务（架构设计、安全审计、难题攻坚）：使用 Opus 模型，保证质量

**配置示例**：
```json
// .claude/settings.local.json
{
  "model": "claude-haiku-4",
  "overrides": {
    "/code-review": {"model": "claude-sonnet-4-6"},
    "/security-review": {"model": "claude-opus-4-6"}
  }
}
```

### 策略2：批量处理
**不推荐：逐个处理**
```bash
claude "检查src/a.ts的代码质量"
claude "检查src/b.ts的代码质量"
claude "检查src/c.ts的代码质量"
```
成本：3次独立调用，约$0.18

**推荐：批量处理**
```bash
claude "检查src/目录下所有.ts文件的代码质量，生成汇总报告"
```
成本：1次调用，约$0.06，节省2/3成本

### 策略3：优化上下文
- 定期清理对话历史：`/clear` 命令减少不必要的上下文tokens
- 精简CLAUDE.md：删除不必要的内容，保持<5000字符
- 使用文件引用：不要在对话中粘贴大段代码，引用文件路径
- 分解大任务：将复杂任务拆分为多个小任务，每次只传递必要上下文

### 策略4：缓存复用
启用缓存功能，相同请求不需要重复调用API：
```json
// .claude/settings.json
{
  "cache": {
    "enabled": true,
    "ttl": 3600,
    "maxSize": "100mb"
  }
}
```

### 策略5：成本监控与预警
```json
// .claude/settings.json
{
  "costControl": {
    "dailyLimit": 10.0,      // 日限额10美元
    "monthlyLimit": 200.0,   // 月限额200美元
    "alertThreshold": 0.8,   // 使用80%时告警
    "alertEmail": "admin@company.com",
    "actions": {
      "onDailyLimitReached": "warn",   // 达到日限额时警告
      "onMonthlyLimitReached": "block" // 达到月限额时阻止使用
    }
  }
}
```
