# 个人技术博客 — 需求文档

**项目名称**：个人博客  
**文档版本**：v1.0  
**日期**：2026-05-12  
**定位**：每日写文章、记录技术成长、分享个人思考

---

## 一、项目定位与目标

### 1.1 核心场景

"我每天写文章，需要一个平台发布和展示它们。"

- **写作频率**：每日 1 篇，长期积累
- **内容类型**：技术文章、读书笔记、生活感悟、项目总结
- **读者**：技术同行、朋友、潜在雇主（面试作品展示）

### 1.2 博客三大核心价值

| 价值 | 说明 |
|------|------|
| **阅读体验** | 文章呈现清晰、排版舒适、代码可读 |
| **写作效率** | 后台写文章快速，Markdown 支持，自动保存 |
| **内容发现** | 分类、标签、搜索，帮助读者找到感兴趣的文章 |

---

## 二、网站结构（信息架构）

```
个人博客
├── 首页（Home）               ← 个人介绍 + 最新文章列表
├── 文章列表（Articles）       ← 全部文章，支持筛选和搜索
├── 文章详情（Article/:slug）  ← 阅读体验核心页
├── 分类（Categories）         ← 按分类浏览文章
├── 标签（Tags）               ← 按标签浏览文章
├── 归档（Archive）            ← 按年月时间线
└── 关于我（About）            ← 个人介绍、技术栈、联系方式
```

## 三、各页面需求详述

### 3.1 首页（Home）

#### 区块结构（从上到下）

| 区块 | 内容 | 可配置 |
|------|------|--------|
| **导航栏** | Logo/博客名 + 菜单（首页/文章/分类/归档/关于） + 搜索图标 | 博客名、菜单 |
| **个人简介区** | 头像（圆形）+ 博主名字 + 一句话简介 + 社交链接（GitHub/掘金/微信） | 头像、名字、简介、链接 |
| **精选文章** | 1-2 篇置顶/精选文章（大卡片，有封面图） | 手动指定 |
| **最新文章** | 最新 6-10 篇文章卡片（标题 + 摘要 + 标签 + 日期 + 阅读时长） | 自动 |
| **分类导航** | 所有分类 + 各分类文章数量，横向标签展示 | 自动 |
| **页脚** | 版权信息 + ICP 备案 + 统计 | 可配置 |

---

### 3.2 文章列表页（Articles）

- 搜索框（关键词搜索标题+正文）
- 分类 / 标签筛选（多选）
- 文章排序：最新发布 / 最多阅读 / 最多评论
- 文章卡片：
  - 封面图（无封面时显示分类色块）
  - 标题（最多 2 行）
  - 摘要（最多 3 行，120 字）
  - 分类标签、发布日期、预计阅读时长
- 分页：每页 10 篇

---

### 3.3 文章详情页（核心页面）

这是博客最重要的页面，需要极佳的阅读体验。

#### 布局

```
┌──────────────────────────────────────────────────┐
│ 导航栏（滚动后固定）                              │
├──────────────────────────────────────────────────┤
│                                                  │
│  文章标题（大，居中）                             │
│  发布日期 · 分类 · 标签 · 阅读时长 · 阅读量      │
│  封面图（全宽，可选）                             │
│                                                  │
├───────────────────────────┬──────────────────────┤
│                           │ 目录（TOC）            │
│   文章正文                 │ sticky 固定           │
│   （Markdown 渲染）        │                      │
│                           │ ─────────────         │
│   最大宽度 780px           │ 作者信息卡            │
│   行高 1.9                 │                      │
│   正文字号 17px            │                      │
│                           │                      │
├───────────────────────────┴──────────────────────┤
│  标签云                                           │
│  上一篇 / 下一篇                                  │
│  相关文章（同分类 3 篇）                           │
└──────────────────────────────────────────────────┘
```

#### Markdown 渲染支持

| 元素 | 要求 |
|------|------|
| 标题 H1-H6 | 自动生成锚点（用于 TOC）|
| 段落 | 16-17px，行高 1.9 |
| 代码块（``` ```） | 语法高亮（Prism.js / Shiki），支持语言标注，一键复制 |
| 行内代码 | 背景色高亮，等宽字体 |
| 图片 | 居中，支持 alt 文字说明，点击放大 |
| 引用块 | 左侧竖线，背景浅色 |
| 表格 | 带边框，奇偶行区分 |
| 数学公式 | KaTeX 渲染（可选，二期） |
| 分割线 | 细线 |
| 链接 | 主色，hover 下划线，外链新窗口打开 |
| 有序/无序列表 | 正常缩进 |
| 任务列表 | `- [x]` 支持勾选样式 |

#### 阅读体验细节

- **阅读进度条**：页面顶部细线进度条，随滚动更新
- **代码块**：显示语言名称 + 复制按钮
- **图片 Lightbox**：点击放大查看
- **目录（TOC）**：右侧 sticky，滚动时高亮当前章节
- **返回顶部**：右下角浮动按钮，滚动超过 300px 时出现
- **预计阅读时长**：按 200字/分钟计算

### 3.5 分类与标签页

**分类页**：卡片式展示，每个分类显示名称 + 文章数 + 最近更新时间，点击进入该分类文章列表。

**标签页**：标签云（词云），根据文章数调整字号大小（12-24px 范围），点击进入该标签文章列表。

---

### 3.6 关于我（About）

| 区块 | 内容 |
|------|------|
| 个人照片 + 名字 + 职位 | 居中展示 |

## 四、写文章体验（后台）

这部分决定你每天能否快速完成写作，是**最重要的效率需求**。

### 4.1 文章编辑器需求

| 功能 | 优先级 | 说明 |
|------|--------|------|
| Markdown 编辑器 | P0 | 左侧编辑，右侧实时预览（双栏布局）|
| 自动保存草稿 | P0 | 每 30 秒自动保存，防止内容丢失 |
| 快捷键支持 | P0 | Ctrl+S 保存，Ctrl+B 加粗，Ctrl+I 斜体 |
| 图片拖拽上传 | P0 | 拖入图片自动上传并插入 Markdown 语法 |
| 图片粘贴上传 | P0 | 截图后 Ctrl+V 直接粘贴 |
| 全屏写作模式 | P1 | 隐藏所有干扰，专注写作 |
| 字数统计 | P1 | 实时显示字数、预计阅读时长 |
| 代码高亮预览 | P1 | 编辑器右侧实时渲染代码块 |
| 文章封面设置 | P1 | 上传封面图或从附件库选择 |
| 文章摘要 | P1 | 手动填写或自动截取前 200 字 |
| 定时发布 | P2 | 选择时间后自动发布 |
| 版本历史 | P2 | 保留最近 10 个历史版本，可回滚 |

### 4.2 分类和标签

- 分类：写文章时下拉选择（单选），支持新增
- 标签：输入框 + 回车添加，支持已有标签联想，最多 10 个

### 4.3 写作流程

```
打开后台 → 点击"写文章" 
→ 输入标题（自动生成 slug）
→ 选择分类、添加标签 
→ Markdown 编辑器写作（自动每30秒保存草稿）
→ 设置封面图（可选）
→ 填写摘要（可选，不填则自动截取）
→ 预览效果
→ 点击"发布" → 一键部署到前台
```

## 五、视觉设计规范

### 5.1 博客整体风格

**关键词**：简洁、文艺、阅读友好、有点温度

区别于公司官网的"专业商务"，博客需要：
- 更多白色和留白
- 字体大，行距宽
- 配色柔和不刺眼

### 5.2 色彩方案

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主色（强调/链接） | `#3B82F6` | 清爽蓝，代码感 |
| 正文色 | `#374151` | 深灰，比纯黑柔和 |
| 次要文字 | `#6B7280` | 日期、摘要 |
| 背景 | `#FFFFFF` | 纯白 |
| 卡片背景 | `#F9FAFB` | 极浅灰 |
| 代码块背景 | `#1E293B` | 深色，护眼 |
| 代码高亮 | GitHub Dark 主题 | |
| 边框/分割线 | `#E5E7EB` | |
| 标签背景 | `#EFF6FF` | 浅蓝 |
| 标签文字 | `#2563EB` | |

### 5.3 字体

| 场景 | 字体 | 大小 |
|------|------|------|
| 博客名/标题 | 思源黑体 / PingFang SC | 按层级 |
| 文章正文 | 思源宋体 / Georgia（备选） | 17px |
| 代码 | JetBrains Mono / Fira Code | 14px |

> 正文使用衬线字体（宋体系）可提升长文阅读舒适度，但也可统一用无衬线字体，按个人喜好定。

### 5.4 排版间距

- 文章正文最大宽度：780px，居中
- 段落间距：1em
- 行高：1.9
- 代码块：14px，行高 1.7，左右 padding 20px，上下 16px
- 标题上方间距：2em，下方间距：0.8em

## 六、SEO 需求

- 每篇文章独立 `<title>` = "文章标题 | 博客名"
- `<meta description>` = 文章摘要
- Open Graph 标签（分享到微信/微博显示封面图）：`og:title`, `og:description`, `og:image`
- 文章 URL：`/articles/{slug}`（如：`/articles/vue3-best-practices`）
- `sitemap.xml`：每次发布后自动更新
- 结构化数据：每篇文章添加 `BlogPosting` Schema
