# 网站构建系统 - 极简需求

## 核心理念
**一句话概括**：管理员通过表单配置页面内容，前端根据主题模板自动渲染展示。

---

## 一、系统角色（2个）

1. **管理员** - 在后台配置网站内容
2. **访客** - 在前端浏览网站

---

## 二、核心功能（3个）

### 1. 主题管理
- 系统预置2个主题：博客主题、企业主题
- 管理员选择一个主题作为当前使用的主题
- **就这么简单，不支持自定义主题**

### 2. 页面配置
- 管理员创建页面（如：首页、关于我们、产品列表）
- 填写表单配置页面内容：
  - 页面名称
  - 页面路由（如 /about）
  - 页面配置（JSON格式，根据主题不同字段不同）
- **不需要拖拽，不需要可视化编辑器，就是填表单**

### 3. 内容展示
- 前端根据当前主题 + 页面配置自动渲染
- 访客访问 /about，系统找到对应页面配置，用主题模板渲染出来
- **前端不需要管理功能，只负责展示**

---

## 三、数据结构（3张表）

### 1. themes（主题表）
```
id, name, description, is_active
```
- 只有2条数据：博客主题、企业主题
- is_active 标记当前使用哪个主题

### 2. page_configs（页面配置表）
```
id, theme_id, name, route, config_json, sort_order
```
- config_json 存储页面的所有配置数据
- 不同主题的页面，config_json 结构不同

### 3. media（媒体文件表）
```
id, filename, url, type
```
- 存储上传的图片、视频等文件

## 四、主题配置说明

### 博客主题的页面配置示例
```json
{
  "title": "我的博客",
  "subtitle": "分享技术与生活",
  "articles": [
    {
      "title": "文章标题",
      "summary": "文章摘要",
      "cover": "封面图URL",
      "date": "2024-01-01"
    }
  ]
}
```

### 企业主题的页面配置示例
```json
{
  "banner": {
    "title": "企业标题",
    "description": "企业描述",
    "image": "背景图URL"
  },
  "products": [
    {
      "name": "产品名称",
      "description": "产品描述",
      "image": "产品图URL"
    }
  ]
}
```

## 五、使用流程

### 管理员操作流程
1. 登录后台
2. 选择主题（博客或企业）
3. 创建页面，填写表单配置内容
4. 保存后，前端自动生效

### 访客访问流程
1. 访问网站 URL（如 /about）
2. 系统查找对应页面配置
3. 根据当前主题渲染页面
4. 展示给访客

---

## 六、技术实现要点

### 后端（NestJS）
- 提供主题切换 API
- 提供页面配置的增删改查 API
- 提供媒体文件上传 API
- 提供前端获取页面配置的 API

### 管理后台（Vue3 + Element Plus）
- 主题选择页面（单选框）
- 页面配置列表（表格）
- 页面配置表单（根据主题动态生成表单字段）
- 媒体文件上传组件

### 前端官网（Vue3）
- 根据路由获取页面配置
- 根据主题加载对应的组件模板
- 渲染页面内容

---

## 七、MVP 范围（最小可用版本）

**第一阶段只做这些：**
1. 2个主题（博客、企业）
2. 主题切换功能
3. 页面配置的增删改查
4. 图片上传
5. 前端展示功能

**不做的功能：**
- ❌ 自定义主题
- ❌ 拖拽编辑器
- ❌ 可视化预览
- ❌ 多语言
- ❌ SEO优化
- ❌ 权限管理（只有一个管理员）
- ❌ 数据统计

---

## 八、关键设计原则

1. **内容与展示分离** - 配置存JSON，主题负责渲染
2. **表单配置** - 不做拖拽，降低复杂度
3. **预置主题** - 不支持自定义，减少开发量
4. **单一管理员** - 不做权限系统
5. **极简数据模型** - 只有3张核心表

---

## 九、成功标准

**系统上线后，管理员能够：**
1. 5分钟内切换主题
2. 10分钟内创建一个新页面
3. 不需要懂代码就能配置内容

**访客能够：**
1. 正常访问所有页面
2. 看到管理员配置的内容
3. 页面加载速度快（< 2秒）
