## 一、项目背景与目标

### 1.1 公司简介

山东春昌食品科技有限公司，主营食品加工与科技研发，面向 B 端客户和零售终端。官网是公司对外展示品牌形象、产品实力、公司文化的第一入口，也是潜在客户了解合作意向的重要渠道。

### 1.2 建站目标

| 目标 | 说明 |
|------|------|
| 品牌展示 | 传递专业、可信赖的食品科技企业形象 |
| 产品展示 | 清晰展示主营产品系列，支持图文详情 |
| 招商合作 | 引导客户主动联系，沉淀潜在商机 |
| SEO 获客 | 通过内容结构支持搜索引擎收录，提升自然流量 |
| 内容运营 | 新闻动态、行业资讯，支持后台随时更新 |

### 1.3 目标受众

| 用户类型 | 特征 | 访问诉求 |
|---------|------|---------|
| 采购商/代理商 | B 端，关注产品品质与供应能力 | 看产品、了解实力、找联系方式 |
| 终端消费者 | C 端，偶发访问 | 了解品牌故事、查看产品信息 |
| 求职者 | 关注公司规模与文化 | 了解公司介绍、联系方式 |
| 搜索引擎爬虫 | — | 结构清晰的 HTML，完整 meta 信息 |

## 三、各页面需求详述

### 3.1 首页（Home）

#### 内容区块（从上到下）

| 区块 | 内容 | 可配置项 |
|------|------|---------|
| **导航栏** | Logo + 菜单 + 联系电话 | Logo 图片、菜单项、电话 |
| **英雄区（Hero Banner）** | 全屏背景大图 + 公司标语 + 副标题 + CTA 按钮 | 背景图、标语、副标题、按钮文字和链接 |
| **核心优势** | 3-4 个优势卡片（图标 + 标题 + 描述） | 各卡片内容 |
| **产品精选** | 4-6 个主推产品卡片（图片 + 名称 + 简介） | 产品列表 |
| **关于我们简介** | 左图右文 or 左文右图，带"了解更多"按钮 | 图片、文字、按钮 |
| **数字成就** | 成立年份、服务客户数、产品种类、获奖数量 | 各项数字和标签 |
| **新闻动态** | 最新 3 条新闻（标题 + 日期 + 摘要） | 自动拉取最新新闻 |
| **合作品牌/资质展示** | 横向滚动的合作伙伴 Logo 或资质证书缩略图 | 图片列表 |
| **页脚（Footer）** | 公司名称、地址、电话、邮箱、ICP 备案、友情链接、版权 | 各联系信息 |

#### 技术要求
- Hero Banner 支持视差滚动（Parallax）效果
- 数字成就区块：进入视口时数字滚动动画（countup）
- 页面加载：核心内容优先渲染，图片懒加载

---

### 3.2 关于我们（About）

#### 区块结构

1. **页面横幅**：大标题"关于春昌" + 面包屑
2. **公司简介**：500 字以内图文混排，支持富文本
3. **发展历程**：纵向时间轴，每个节点：年份 + 事件标题 + 描述
4. **企业资质荣誉**：证书/奖项图片展示（Grid 布局，支持 Lightbox 放大）
5. **团队介绍**（可选）：核心团队成员卡片

#### 可配置字段

```json
{
  "banner": { "title": "关于春昌", "bg": "图片URL" },
  "intro": {
    "text": "公司简介富文本",
    "image": "公司照片URL"
  },
  "milestones": [
    { "year": "2010", "title": "公司成立", "desc": "在山东济南注册成立..." }
  ],
  "honors": [
    { "image": "证书图片URL", "title": "国家高新技术企业认证" }
  ]
}
```

### 3.3 产品中心（Products）

#### 产品列表页

- 顶部分类标签筛选（全部 / 分类1 / 分类2 / ...）
- 产品卡片 Grid（桌面端 3 列，平板 2 列，手机 1 列）
- 每个卡片：封面图 + 产品名称 + 简短描述 + "查看详情"按钮
- 支持分页（每页 12 条）

#### 产品详情页

| 内容 | 描述 |
|------|------|
| 产品主图 | 支持多图轮播 |
| 产品名称 | 大标题 |
| 产品规格 | 表格展示（净含量、保质期、储存条件等） |
| 产品描述 | 富文本，支持图文 |
| 产品特点 | 带图标的要点列表 |
| 相关产品推荐 | 同分类下的其他产品 3-4 个 |

#### 可配置字段

```json
{
  "category": "产品分类名",
  "name": "产品名称",
  "images": ["主图URL", "图2URL"],
  "spec": [
    { "label": "净含量", "value": "500g" },
    { "label": "保质期", "value": "12个月" }
  ],
  "description": "富文本内容",
  "features": ["特点1", "特点2"]
}
```

## 四、视觉设计规范

### 4.1 品牌色彩

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主色 | `#1B6B3A` | 深绿色——食品/自然/安全感 |
| 辅助色 | `#F5A623` | 暖橙色——活力/食欲感 |
| 深色文字 | `#1A2332` | |
| 次要文字 | `#5C6B7D` | |
| 背景浅灰 | `#F7F8FA` | |
| 白色 | `#FFFFFF` | |
| 成功/绿色 | `#22C55E` | |

**色彩理念**：深绿主色传递食品安全、自然健康的品牌价值；橙色辅助色用于 CTA 按钮和重点信息高亮，增加食欲联想与行动引导。

### 4.2 字体

| 场景 | 字体 | 大小 | 字重 |
|------|------|------|------|
| 大标题（Hero） | 微软雅黑 / Source Han Sans | 48-64px | 700 |
| 页面标题 H2 | 微软雅黑 | 32-40px | 600 |
| 区块标题 H3 | 微软雅黑 | 24px | 600 |
| 正文 | 微软雅黑 | 16px | 400 |
| 辅助文字 | 微软雅黑 | 14px | 400 |
| 说明文字 | 微软雅黑 | 13px | 400 |

### 4.3 图片规范

| 类型 | 尺寸 | 格式 | 大小限制 |
|------|------|------|---------|
| Hero 背景图 | 1920×1080 | WebP/JPG | ≤ 400KB |
| 产品主图 | 800×800 | WebP/JPG | ≤ 200KB |
| 新闻封面 | 800×450 | WebP/JPG | ≤ 150KB |
| 合作伙伴 Logo | 200×80 | PNG（透明） | ≤ 30KB |
| 荣誉证书 | 600×800 | JPG | ≤ 150KB |

### 4.4 响应式断点

| 设备 | 断点 | 适配策略 |
|------|------|---------|
| 桌面端 | ≥ 1200px | 全功能布局 |
| 平板横屏 | 1024-1199px | 轻微压缩 |
| 平板竖屏 | 768-1023px | 二列布局 |
| 手机端 | < 768px | 单列，汉堡菜单 |

### 4.5 动效规范

| 场景 | 效果 | 时长 |
|------|------|------|
| 页面首屏加载 | 标题 + 副标题从下 fadeInUp | 0.6s |
| 区块进入视口 | 从下 fadeInUp，stagger 0.1s | 0.5s |
| 按钮 hover | 背景变深 + 轻微上移 | 0.2s |
| 产品卡片 hover | 阴影加深 + 封面图缩放 1.05 | 0.3s |
| 数字成就 | countUp 动画 | 1.5s |
| 导航栏滚动 | 透明 → 白色背景 + 阴影 | 0.3s |

## 五、SEO 需求

- 每个页面独立 `<title>` 和 `<meta description>`（后台可配置）
- 页面 URL 语义化：`/about`、`/products/123`、`/news/456`
- 图片 `alt` 属性必填
- 结构化数据（Schema.org）：
  - 首页：`Organization`
  - 产品页：`Product`
  - 新闻页：`NewsArticle`
- `sitemap.xml` 自动生成
- `robots.txt` 配置
- 全站 HTTPS

## 六、性能要求

| 指标 | 目标值 |
|------|--------|
| 首屏加载时间（LCP） | ≤ 2.5s（移动端） |
| 交互响应时间（FID） | ≤ 100ms |
| 累积布局偏移（CLS） | ≤ 0.1 |
| Lighthouse 评分 | ≥ 85（移动端），≥ 90（桌面端） |
| 图片格式 | 优先 WebP，支持 JPG fallback |
