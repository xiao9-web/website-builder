## 1. 项目一句话定位

**一个用 Vue3 + NestJS 自建的多站点内容管理系统，主场是个人技术博客，副场能快速克隆出独立域名的企业官网站点。**

- 「多站点」：**一套后台 / 一套数据库 / N 个前端站点实例**，每个站点有独立域名、主题、菜单、内容边界
- 「主场」：个人博客（已 70% 完成，是首要交付）
- 「副场」：家人企业官网作为独立站点（第二优先，走同一架构）

## 2. 产品目标（按优先级）

| 优先级 | 目标 | 完成判据 |
|---|---|---|
| P0 | 个人技术博客能上线对外发布文章 | 自有域名可访问；文章发布/分类/标签/搜索全链路跑通 |
| P0 | 作为求职面试作品可完整演示 | 在 `docs/INTERVIEW_TALKTRACK.md` 中能讲清 5 个技术亮点；代码质量经得起 code review |
| P1 | 多站点架构能支撑第二个站点（家人企业官网） | 同一 admin 能切换"当前编辑站点"；前端可按站点 ID 加载独立配置 |
| P2 | 家人企业官网上线 | 独立域名访问；首页含公司介绍、业务、联系方式 |

## 3. 目标受众

| 受众 | 关心什么 | 本项目如何服务 |
|---|---|---|
| **博客读者**（技术同行） | 文章内容质量、阅读体验、代码高亮 | 前台文章详情页、目录导航、相关推荐、响应式 |
| **博主本人**（内容生产者） | 写作效率、草稿管理、SEO | admin 富文本编辑、自动保存、回收站、SEO 字段 |
| **面试官** | 架构合理性、技术深度、工程素养 | 多站点架构、版本回滚、权限系统、可视化编辑器自研 |
| **家人企业访客**（次要） | 公司信息、联系方式 | 副站点首页 + 简介 + 联系页（最小可用） |

**非受众**：非技术小白用户（不面向他们做零代码建站器），企业内部多人协作团队（不做多租户/工作流）。

## 4. Non-Goals（明确不做的事）

以下能力曾出现在旧文档中，**本版本明确不做**，避免再被列入迭代：

1. **模板市场 / 组件市场**：不做"可供他人选购的"模板商店。自研 5 个基础组件（已有）够用
2. **给非技术人员的零代码建站器**：不做面向小白的完整拖拽建站产品
3. **阿里云 OSS 一键部署 + CDN 自动刷新**：不做。部署走传统静态构建 + Nginx / 对象存储直传，不再追求自动化闭环
4. **多主题市场、暗黑模式切换、Google Fonts 集成**：不做。主题通过站点配置表手工切换
5. **定时发布、发布审核流、邮件通知**：不做（P2 backlog，当前版本砍）
6. **Disqus/Gitalk 第三方评论集成**：不做（自建评论成本高，关闭即可）
7. **ECS/函数计算/Docker Swarm 部署编排**：不做。docker-compose 本地起 MySQL 足够

## 5. 技术栈锁定

| 层 | 技术 | 理由 |
|---|---|---|
| 前台官网 | Vue3 + Vite + Tailwind + 自研 ComponentRenderer | 已落地 |
| 后台管理 | Vue3 + Element Plus + Pinia + WangEditor + 自研 VisualEditor | 已落地 |
| 后端 | NestJS 10 + TypeORM + MySQL 8 | 已落地 |
| 认证 | JWT + Passport + RBAC（3 角色） | 已落地 |
| 文件存储 | 本地磁盘 `uploads/YYYY/MM`（OSS 依赖保留但不启用） | 已落地 |
| 部署 | 静态构建 + Nginx | 新定（砍 OSS 一键部署） |

锁定含义：以上是本版本的最终选型，不再因"面试亮点"引入 Redis/ES/K8s 等超纲技术。
